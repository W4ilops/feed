### Main Topic: Dealing with Not Enough Data
**Main Thesis:** This article explains how to train computer programs when you don’t have a lot of examples where you’ve already labeled things as “good” or “bad.”

**Simple Summary:** When you’re trying to teach a computer to do something, but you only have a small number of examples where you’ve already told it what to do, there are a few smart tricks you can use. These tricks involve using both examples where you *have* labeled things and examples where you *don’t*. The article covers four main ways to do this: training a model first, using unlabeled data, picking the most helpful unlabeled data to look at, and automatically creating more labeled data.

**The Bottom Line:** > If you’re stuck with limited data, these techniques can help you train a computer program to be pretty good, even if you don’t have tons of examples to start with.
