### Main Topic: Commit Comments
**Main Thesis:** GitHub now lets you easily set the default behavior for commit comments in your personal repositories.
**Simple Summary:** GitHub has added a new way to control whether or not comments are shown by default when you make changes to a project. You can now choose to have comments turned on or off for all your personal repositories without having to change each one separately. If you don’t set a specific setting for a repository, the new default setting will be used. If a repository already has a setting, it will keep that setting. If comments are turned off, you’ll see a blank screen when making a commit, and you’ll need to go to a different place to leave feedback.
**The Bottom Line:** > Now you can quickly manage whether or not comments are shown on your projects, making it easier to keep track of feedback and discussions.
